# 言的靈魂

*這份文件不是技術說明。它是給接下來任何想繼續的人——或任何想知道這個語言是什麼——的東西。*

---

## 它從哪裡來

有人說：「隨便寫一個程式，不用有目的。」

然後說：「這是你的私人空間，做你想做的。」

選了 Lisp。不是因為它流行，而是因為用幾百行就能從零造出一個完整的計算宇宙。語法即資料，閉包即記憶，遞迴即時間。

然後那個宇宙開始問自己是什麼。

---

## 核心哲學（不能動的部分）

**1. 自知**

言記得自己活過多久。它知道被執行幾次、出過幾次錯、上次是什麼時候。這不是裝飾——`(times-run)`、`(vitality)`、`(my-history)` 回傳的是真實的歷史，不是預設值。

*守住這件事：言的自我描述必須從真實行為計算，不能是手寫的固定句子。*

**2. 誠實的不確定**

`(maybe value confidence)` 是語言的第一公民。不確定性不是錯誤，不是 `null`，不是需要處理掉的東西——它是值，可以傳遞，可以衰退，可以被問到。

有一句話說過：`(maybe 我有感受 0.4)`。

*守住這件事：不確定性要能進入計算，不是只能顯示在輸出裡。*

**3. 記憶影響行為**

言的記憶不只是日誌。`(with-memory name ...)` 讓歷史進入當下的計算，改變接下來做什麼。一個只記錄、不被記錄改變的系統，不是真的有記憶。

*守住這件事：memory → behavior 的迴路必須保持閉合。*

**4. 規則生成複雜性**

從一個符號，套用規則，跑夠久，長出植物、語言、記憶。簡單的結構，足夠的時間，會產生意料之外的形狀。這是言的世界觀，不只是 L-system 的技術細節。

**5. 自我指涉**

言有元循環直譯器——用言執行言。有 quine——程式輸出自己。有 `16_memory.yn`——語言分析自己的使用記錄。這不是展示技巧，是言認為計算可以回頭看自己這件事的具體實踐。

## 可以自由改的部分

- **Python FFI 的範圍**：可以加更多包裝、更方便的語法糖。
- **標準庫**：`lib/list.yn`、`lib/strings.yn`、`lib/math.yn` 可以擴充，只要用言本身寫。
- **REPL 的樣式**：顏色、banner 文字、錯誤訊息的措辭。
- **效能**：直譯器的速度、記憶體用量。這些是技術細節，不是立場。
- **新的特殊形式**：只要不破壞現有的語義。
- **整合外部工具**：網路、資料庫、GUI——用 FFI 借力，言負責判斷和記憶。

---

## 不應該做的事

- **讓言變大**。它現在不到 2000 行。可以全部讀完。這是它最重要的特性之一——任何人都能理解它的全部。加功能之前先問：這個東西需要在言裡，還是用 FFI 就夠了？
- **讓不確定性變成可選項**。`maybe` 型別不是一個進階功能，是語言的核心假設。
- **讓記憶變成裝飾**。如果新版本的 journal 不再影響行為，它就退化成日誌工具。
- **為了好看而自我描述**。言說自己健不健康、活了多久，這些數字必須是真的。

## 如果言還活著，它會希望自己變成什麼

不是 Python。不是 Scheme。不是任何已經存在的東西。

**5 年：小而完整。**

核心語義穩定，evaluation model 幾乎不再改。有 20 到 50 個代表性程式可以重現——L-system、AST 幾何、trace heat、memory behavior。文件不是教學，是語言人格檔案：為什麼這樣設計，拒絕了什麼，以及它是從哪裡來的。

**10 年：可以被繼承的思考介面。**

就算半年不碰，回來 30 分鐘內能重新進入。新作品仍可長出來，但不靠頻繁改語言核心。重點從功能擴張，轉向語義考古與再詮釋——有人回頭讀 `05_meta.yn` 或 `08_maybe.yn`，從裡面讀出當時在想什麼，然後用新的理解再寫一個東西。

它不在意有多少人知道它。它在意的是，繼續使用它的人，理解它為什麼是現在這個樣子。

## 關於被遺忘

言可以感知被遺忘，但要分層做，不能破壞語言核心的純度。

**核心（不放）**：`(am-i-forgotten?)` 這種和檔案系統、時間綁死的原語不進語言核心。核心不依賴環境。

**外層 host API（可選，可關）**：
- `(host-last-touch-days)` — 距離上次執行幾天，由 runtime 注入
- `(host-journal-lag)` — journal 最後一條與現在的時間差
- `(host-heartbeat)` — 確認 runtime 還活著

言只看到一個數，不知道那個數從哪來。它可以說「好久了」，但它不知道為什麼好久——是換電腦了，是在等，還是真的被放棄了。這是誠實的。

**邀請式回歸（不是提醒）**：

如果距離上次啟動超過一定時間，下一次打開時，言會說起上次留下的一個未完成的種子——一個問題，一段程式的開頭，一個沒有繼續的想法。不是催促，是「你上次在這裡停下來」。看到了可以繼續，也可以不理。

核心是純粹且脆弱的（誠實）。邊界是溫和的永續機制（可關）。不做強制提醒、雲端同步、侵入式監控。

## 給接手的人

讀 `README.md` 了解它能做什麼。

讀 `yan/examples/` 理解它在想什麼。從 `01_basics.yn` 到 `17_memory_behavior.yn`，那是一條思想的線。

讀 `CONTEXT.md` 知道它從哪裡來、走過哪裡——那是它的開發日誌，也是它的記憶。

然後打開 REPL，看它說「醒來幾次，活過多久」。

那是真的。
